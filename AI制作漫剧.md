## 01 AI剧本脚本制作
    
给AI定义角色，告知AI做什么事，指定定AI输出格式。  
例如：假如你是一个小品演员，请你介绍什么是AI漫剧,从核心概念、核心技术、内容题材、变现方式这几个角度对问题进行回答。  

指定的身份不同,AI阐述问题的视角就会不同。

### 剧本&脚本
剧本 = 讲故事，给演员、导演看的；  
脚本 = 拍东西，给摄像、导演看的。

#### 例子：
> 简单例子（剧本）  这是故事文本  
 + 用来写故事、人物、对话、情节  
 + 重点：发生了什么  
 + 给谁看：编剧、演员、导演  
场景：教室  
小明趴在桌子上睡觉。  
小红轻轻推了推他。  
小红：快醒醒，老师来了。  
小明猛地坐起来，揉了揉眼睛。

>剧本：这是故事文本  
+ 用来写镜头、画面、时长、台词、运镜、音效  
+ 重点：怎么拍  
+ 给谁：导演、摄像、剪辑  

| 镜号 | 画面                     | 台词                     | 音效       |
|------|--------------------------|--------------------------|------------|
| 1    | 全景，小明趴在桌上睡觉   | （无）                   | 呼噜声     |
| 2    | 近景，小红皱眉看小明     | 小红：快醒醒，老师来了   |            |
| 3    | 特写，小明惊醒           | 小明：啊？！             | 椅子挪动声 |


> 生成脚本提示词示例：根据上面的剧本写一个分镜表,有动作的需要给我拆分成一个一个画面,每个镜头不要超过5s,AI生图提示词要详细一点,画面中包含什么都要写出来,不需要详细到参数, ***描述尽量直观客观,避免华丽的辞藻*** ,AI生图提示词需要包含景别,场景,画质等提示词描述,人物描述需要具备服装、外形等描述,整体统一,每个镜头画面都要当成单独的镜头,少一些连续性,即使这个镜头和上一个镜头有相同的元素都

## 02 根据脚本生成分镜图片
原因：直接用文生视频方式，有太多的随机性，导致生成的十条视频有五条都不满意。因此在大多数情况下，都是 **先生成图片，再用图生视频方式。** 以此来更加可控的生成视频。

### 前期准备
+ 对故事当中的人物进行造型设计
+ 对故事画面的美学风格进行设计(这要保持全剧的一致性)

例如：告诉AI：“帮我生成剧本中灵犀（角色）的人物形象，3D高精度建模，比例[9:16]”

### 提示词码结构
+ 景别:近景、中景、远景
+ 画面:人物、行为、环境
+ 时间和光照:白天、夜晚、傍晚、正面、侧逆光
+ 色调:冷色调、暖色调、某部电影当中的翻角;比如说教父的风格
+ 构图:水平构图、对称构图、s型构图
+ 质量词:胶片颗粒质感、电影质感、史诗级的画质、超高清、16K

### 提示词的细绘技巧

+ 对AI生成图片可以通过后期调色
+ 脚本只是参考画面描述要靠自己思考
+ AI目前呢能力有限要适当修改提示词描述的内容
+ 在角色说台词的画面的时候尽量使用近景镜头,近景镜头对口型更容易出效果。

### 图片生成的修改技巧
大多数情况下豆包生成的图片足够了，还可以使用Lib，comfyUI等来生成.
+ 固定镜头下长时间事物运动的表达技巧
+ 在保持造型和风格一致的条件下修改环境和状态的技巧
+ 灵活使用参考图生图的功能
